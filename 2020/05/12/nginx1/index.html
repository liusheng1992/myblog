Nginx1.16.1的安装
发表于 2020-05-12 | 更新于 2020-07-16
Web服务器
Nginx

官网下载

  • nginx官网：http://nginx.org/

上传服务器

安装依赖环境

gcc

  • 解析 c++

PCRE

  • 解析正则表达式

zlib、zlib-devel

  • 压缩包与解压缩包依赖

openssl

  • SSL安全的加密的套接字协议层，用于HTTP安全传输，也就是https

解压

  • 解压后得到的是源码，需要编译后才能安装

编译前准备

  • 编译之前，先创建nginx临时目录，如果不创建，在启动nginx的过程中会报错
  • 在nginx解压目录，输入如下命令进行配置，目的是为了创建makefile文件

提示

注：\ 代表在命令行中换行，用于提高可读性

配置命令：

命令 解释
–prefix 指定nginx安装目录
–pid-path 指向nginx的pid
–lock-path 锁定安装文件，防止被恶意篡改或误操作
–error-log-path 错误日志
–http-log-path http日志
–with-http_gzip_static_module 启动gzip模块，在线实时压缩输出数据流
–http-client-body-temp-path 设定客户端请求的临时目录
–http-proxy-temp-path 设定http代理临时目录
–http-fastcgi-temp-path 设定fastcgi临时目录
–http-uwsgi-temp-path 设定uwsgi临时目录
–http-scgi-temp-path 设定scgi临时目录

编译

提示

在nginx解压的目录进行make

安装

启动

  • 进入/usr/local/nginx/sbin目录启动nginx

提示

停止：./nginx -s stop

优雅的停止：./nginx -s quit

检测配置文件是否正常：./nginx -t

重新加载：./nginx -s reload

验证

  • 打开浏览器，访问虚拟机所处内网ip即可nginx默认页面

提示

服务器的公网ip:80即可访问

  • TIPS：有些利用虚拟机安装nginx， 宿主机上根本访问不到，必须打开防火墙
命令 含义
–zone 作用域
–add-port=80/tcp 添加端口，格式为：端口/通讯协议
–permanent 永久生效，没有此参数重启后失效

注意事项

  1. 如果在云服务器安装，需要开启默认的nginx端口：80

  2. 如果在虚拟机安装，需要关闭防火墙

  3. 本地win或者mac需要关闭防火墙
